
# what is BDD?
-> Behaviour driven dev 
-> before coding the project, there is the collaboration between business and technical teams to define the behaviour of the system.
-> generates a commin documentation from collaborationwhich is understood by every team including stakeholders.
-> to enhance any features, Userstory is created.


# so what is userstory?? 
->It is the informal explanation of general features from the perspective of end users and how system should behave.
-> examples are created and examples are documented that can be developed and tested with automation.
-> now comes coding phase, to implement user behaviour as per documentation.

# Three phases of BDD:
1. Discovery:         Explore and discuess on how system should behave ie [agreed behaviour of the system]
2. Formulation:       create concrete example that can be automated [documented examples and acceptance test]
        Documented exmmaples also called living documentations.
4. automation:        coding and creating automation tests taking these documentations.


 #  Cucumber? 
   -> tool that facilitates BDD to automate documentation
   -> is a tool that understand your documentation and converts it into automation.
